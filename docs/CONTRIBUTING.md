# Contributing

Thanks for considering a contribution.

## Guidelines
- Keep the code dependency-free unless a strong case is made.
- Prefer small, focused pull requests.
- Add or update documentation when behavior changes.
- Include tests when adding parsing or routing logic.

## Workflow
1. Create a feature branch.
2. Make incremental commits with clear messages.
3. Run `cargo fmt` and `cargo clippy` before opening a PR.
